    Create the directory `.claude/skills/retag/` and write `SKILL.md`.

    YAML frontmatter block at top of file:
    - name: loom:retag
    - description: Rename or split a tag across all documents in the knowledge base
    - argument-hint: "&lt;old-tag&gt; &lt;new-tag&gt; [second-new-tag]"
    - disable-model-invocation: true
    - allowed-tools: Read, Write, Glob, Bash

    Body -- steps for the skill to follow:

    Step 1 (PARSE ARGUMENTS):
    Parse $ARGUMENTS. Split on whitespace:
    - If 2 tokens: MERGE MODE -- replace old-tag with new-tag
    - If 3 tokens: SPLIT MODE -- replace old-tag with both new-tag-1 AND new-tag-2 (add both to tags array)
    - If fewer than 2 tokens: print "Usage: /loom:retag &lt;old-tag&gt; &lt;new-tag&gt; [second-new-tag]" and stop.

    Normalize the new tag(s) immediately: tag.toLowerCase().trim().replace(/\s+/g, '-')
    Print: "Mode: MERGE" or "Mode: SPLIT" and the normalized new tag(s).

    Step 2 (FIND AFFECTED FILES):
    Glob all *.md files in ai-tools-and-services/, cloud-ai-platforms/, companies/.
    For each file: Read it. Check if old-tag appears in the tags frontmatter array.
    Build a list of affected files (files that contain old-tag).
    Print: "Found N files containing tag: &lt;old-tag&gt;"
    If N is 0: print "No documents use tag '&lt;old-tag&gt;'. Nothing to do." and stop.

    Step 3 (UPDATE EACH FILE):
    For each affected file, use Bash with an inline node script using gray-matter to update the tags:
    ```bash
    node -e "
    const matter = require('./node_modules/gray-matter');
    const fs = require('fs');
    const raw = fs.readFileSync('FILEPATH', 'utf8');
    const parsed = matter(raw);
    const oldTag = 'OLD_TAG';
    const newTags = ['NEW_TAG_1']; // or ['NEW_TAG_1', 'NEW_TAG_2'] for split mode
    // Remove old tag and add new tag(s), deduplicate
    parsed.data.tags = [...new Set(
      parsed.data.tags.flatMap(t => t === oldTag ? newTags : [t])
    )];
    fs.writeFileSync('FILEPATH', matter.stringify(parsed.content, parsed.data));
    "
    ```
    Substitute actual file path, old tag, and new tag(s) into the script for each file.
    Print the file path after each successful update.

    Step 4 (REPORT):
    Print a summary:
    - "Updated N files: replaced '&lt;old-tag&gt;' with '&lt;new-tag&gt;'" (or both new tags in split mode)
    - List the modified file paths
    - "Run /loom:deploy to re-embed and publish changes."

    Critical guardrails (must appear verbatim):
    - "Normalize new tag(s) before writing: tag.toLowerCase().trim().replace(/\\s+/g, '-')"
    - "Use gray-matter for frontmatter rewriting to handle all YAML formats correctly."
    - "Deduplicate tags after replacement."
