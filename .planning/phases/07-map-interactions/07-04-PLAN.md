    **1. Build sorted unique dates array and configure slider:**
    ```javascript
    // Build sorted unique date strings from corpus
    const uniqueDates = [...new Set(docs.filter(d => d.date).map(d => d.date))].sort();
    const slider = document.getElementById('map-timeline');
    slider.max = uniqueDates.length > 0 ? String(uniqueDates.length - 1) : '0';
    slider.value = '0';
    // Initialize timelineDate to null (no filter) — slider at position 0 does NOT activate filter
    // Filter only activates once user drags (see input handler below)
    ```

    **2. Slider input handler:**
    ```javascript
    slider.addEventListener('input', () => {
      const idx = parseInt(slider.value, 10);
      if (uniqueDates.length === 0) {
        timelineDate = null;
      } else {
        timelineDate = new Date(uniqueDates[idx]);
      }
      redraw();
    });
    ```
    NOTE: Moving the slider to position 0 activates the filter at the earliest date — this is intentional and correct for timeline scrubbing. If the user wants no filter, they leave the slider at position 0 before touching it (timelineDate is null on initial load). The play button resets to no-filter state when stopped.

    Actually — simpler UX: let initial position of slider mean "no filter"; only activate filter when user has dragged. Track whether user has interacted with slider via a boolean `timelineActive`:
    ```javascript
    let timelineActive = false;
    slider.addEventListener('input', () => {
      timelineActive = true;
      const idx = parseInt(slider.value, 10);
      timelineDate = uniqueDates.length > 0 ? new Date(uniqueDates[idx]) : null;
      redraw();
    });
    ```
    Update gaussianOp to check timelineActive (or just check timelineDate which is still null if not active — fine as-is since timelineDate starts as null and is only set in the input handler).

    **3. Play/pause button:**
    ```javascript
    const playBtn = document.getElementById('map-play');

    function stopPlay() {
      if (playInterval) {
        clearInterval(playInterval);
        playInterval = null;
      }
      playBtn.innerHTML = '&#x25B6;'; // ▶
      playBtn.classList.remove('playing');
    }

    function tick() {
      const max = parseInt(slider.max, 10);
      const val = parseInt(slider.value, 10);
      if (val >= max) {
        stopPlay();
        // Reset filter after play completes
        timelineDate = null;
        redraw();
        return;
      }
      slider.value = String(val + 1);
      slider.dispatchEvent(new Event('input')); // triggers the input handler → redraw()
    }

    playBtn.addEventListener('click', () => {
      if (playInterval) {
        stopPlay();
      } else {
        // Reset slider to start before playing
        slider.value = '0';
        timelineDate = null;
        playInterval = setInterval(tick, 100); // 100ms per step ≈ 10fps scrub
        playBtn.innerHTML = '&#x23F8;'; // ⏸
        playBtn.classList.add('playing');
        tick(); // fire first step immediately
      }
    });

    // Cleanup on page navigation (pitfall 5 from RESEARCH.md)
    window.addEventListener('pagehide', stopPlay);
    ```

    IMPORTANT: The `tick()` function calls `slider.dispatchEvent(new Event('input'))` which fires the slider's input handler, which sets `timelineDate` and calls `redraw()`. This means the play button drives the full filter pipeline correctly without any duplicate redraw logic.

    When play completes (val >= max), reset `timelineDate = null` and call `redraw()` to return all dots to unfiltered state.
