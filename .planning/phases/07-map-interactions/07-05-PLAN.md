<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Human visual verification of all 6 map interactions</name>
  <what-built>
    Complete Phase 7 interaction layer on the /map page:
    - MAP-03: Side panel slides in on dot click with title, tags, summary, article link
    - MAP-04: 5 nearest-neighbor lines drawn on canvas from clicked dot
    - MAP-05: Tag filter buttons dim non-matching dots to ~20% opacity
    - MAP-06: Fuse.js search dims non-matching dots in real time
    - MAP-07: ANY/ALL toggle changes which dots glow with multiple tags selected
    - MAP-08: Timeline slider applies Gaussian opacity fade by document date
    - MAP-09: Play button auto-advances timeline from first to last date, then stops
    - MAP-10: All three filters compose simultaneously
  </what-built>
  <how-to-verify>
    Start local dev server: `cd /home/daniel/codes/loom && npx wrangler pages dev`
    Visit: http://localhost:8788/map

    Verify each interaction in order:

    **MAP-03 — Side panel:**
    1. Click any dot on the canvas
    2. Expected: A panel slides in from the right with the document's title, tags as pills, summary text, and an "Open article →" link
    3. Click the × button — panel closes
    4. Click the same dot again — panel opens again
    5. Click the same dot while panel is open — panel closes (toggle behavior)

    **MAP-04 — Nearest-neighbor lines:**
    1. Click a dot
    2. Expected: 5 thin gray lines draw from the clicked dot to the 5 nearest other dots
    3. Click a different dot — lines move to the new dot's neighbors
    4. Toggle same dot — lines disappear

    **MAP-05 — Tag filter:**
    1. Click any tag button in the controls bar
    2. Expected: Dots whose documents have that tag glow at full opacity; all others dim to ~20% opacity
    3. Click the same tag button again — all dots return to full opacity

    **MAP-07 — ANY/ALL toggle:**
    1. Click two different tag buttons (both active/glowing)
    2. Note which dots are glowing (ANY mode = union)
    3. Click the ANY button to switch to ALL
    4. Expected: Fewer dots glow (only docs that have BOTH selected tags)
    5. Click ALL to switch back to ANY — more dots glow again

    **MAP-06 — Search:**
    1. Clear all tag filters first
    2. Type a word in the search box (e.g. "google" or "aws")
    3. Expected: Matching dots glow; non-matching dims in real time as you type
    4. Clear the search box — all dots return to full opacity

    **MAP-08 — Timeline slider:**
    1. Clear all tag/search filters
    2. Drag the timeline slider left and right
    3. Expected: A Gaussian fade applies — dots near the selected date glow, dots far from it fade
    4. Drag to opposite extreme — a different set of docs is highlighted

    **MAP-09 — Play button:**
    1. Click the ▶ play button
    2. Expected: Slider advances automatically; Gaussian opacity shifts as timeline advances; button changes to ⏸
    3. Let it run to completion — slider stops at end; all dots restore to unfiltered state
    4. Press ⏸ mid-play to pause

    **MAP-10 — Composed filtering:**
    1. Select one tag filter button
    2. Type a partial search term that matches some (not all) of the tag-filtered docs
    3. Drag the timeline slider to a date not matching all remaining docs
    4. Expected: Only docs passing all three filters glow at full opacity; failing any one dims them
  </how-to-verify>
