    ```yaml
    ---

    # Organize Skill

    Audit the Loom knowledge base for tag inconsistencies and suggest fixes. Do NOT modify any files automatically.

    ## Tag normalization rule

    Canonical form: `tag.toLowerCase().trim().replace(/\s+/g, '-')`

    A tag is non-canonical if it contains uppercase letters, leading/trailing whitespace, or spaces instead of hyphens.
    Examples of non-canonical tags: "LLM", "Machine Learning", " api ", "Open Source"
    Canonical equivalents: "llm", "machine-learning", "api", "open-source"

    ## Steps

    1. Use Glob to find all `*.md` files in:
       - `ai-tools-and-services/`
       - `cloud-ai-platforms/`
       - `companies/`
    2. For each file, read it and extract the `tags:` frontmatter field.
    3. Check each tag against the normalization rule. Flag tags that are non-canonical.
    4. Identify tags that appear to refer to the same concept but are spelled differently across files (e.g., "llm" vs "large-language-model" — note as a potential synonym to review, not an error).
    5. Check that each file has at least one tag. Flag files with an empty tags list.
    6. Report findings in a structured table:

       | File | Current Tag | Suggested Fix | Issue Type |
       |------|-------------|---------------|------------|
       | ai-tools-and-services/foo.md | LLM | llm | non-canonical casing |
       | companies/bar.md | (empty) | — | missing tags |

    7. After the table, present a proposed set of edits (show the frontmatter diff for each affected file).
    8. Ask the user: "Apply these fixes? (yes/no)" — do NOT write files unless the user confirms.
    ```

    Create directory `.claude/skills/validate/`.

    Write `.claude/skills/validate/SKILL.md`:
    ```yaml
    ---
    name: validate
    description: Check all Loom documents for frontmatter errors and template non-conformance — report only, no modifications
    disable-model-invocation: true
    allowed-tools: Read, Glob, Bash
    ---

    # Validate Skill

    Validate all documents in the Loom knowledge base. Report all errors. Do NOT modify any files.

    ## Tag normalization rule

    Canonical form: `tag.toLowerCase().trim().replace(/\s+/g, '-')`

    ## Required frontmatter fields

    - `title`: non-empty string
    - `date`: string in YYYY-MM-DD format (regex: `/^\d{4}-\d{2}-\d{2}$/`)
    - `tags`: non-empty list; each tag must be a string

    ## Required document structure (per TEMPLATE.md)

    - H1 heading (`# ...`) that matches the frontmatter `title` exactly
    - `## Summary` section (must appear in document body)
    - `## Content` section (must appear in document body)

    ## Steps

    1. Use Glob to find all `*.md` files in:
       - `ai-tools-and-services/`
       - `cloud-ai-platforms/`
       - `companies/`
    2. For each file:
       a. Parse YAML frontmatter (everything between the first `---` and second `---`).
       b. Check `title` exists and is non-empty.
       c. Check `date` exists and matches YYYY-MM-DD format.
       d. Check `tags` exists, is a list, and has at least one item.
       e. Check each tag value matches the normalization rule (lowercase, hyphen-separated, no spaces).
       f. Find the first `# ` heading in the body. Verify it matches `title`.
       g. Verify `## Summary` appears in the body.
       h. Verify `## Content` appears in the body.
    3. Report all findings in a structured table:

       | File | Field/Section | Error | Severity |
       |------|---------------|-------|----------|
       | ai-tools-and-services/foo.md | date | Missing | error |
       | companies/bar.md | tags[1] | "Open Source" not normalized | warning |
       | cloud-ai-platforms/baz.md | H1 heading | "Baz Service" does not match title "BazService" | error |

    4. End with a summary: "X errors, Y warnings across Z files."
    5. Do NOT modify any files — this skill is read-only.
    ```
