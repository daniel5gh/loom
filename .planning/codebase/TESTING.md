## Quality Assurance (Non-Automated)

Since this project has no automated testing, quality is maintained through:

**Document Template Compliance:**
- All research documents should follow `TEMPLATE.md` structure
- Verify frontmatter fields (title, date, tags) are present and correctly formatted

**Manual Review Checklist:**
- Frontmatter is valid YAML with `title`, `date`, and `tags`
- H1 heading matches frontmatter `title`
- At least one external reference link is included
- File is placed in the correct category directory
- Filename uses lowercase kebab-case

**Potential Automation Opportunities:**
- A Markdown linter (e.g., `markdownlint-cli`) could enforce consistent formatting
- A frontmatter validator script could verify required fields exist
